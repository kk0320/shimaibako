# 自動分類と仮想フォルダ設計

## 目的

写真本体を動かさず、しまい箱内だけで写真を探しやすく分類する。分類結果は `localIdentifier` に紐づけて検索インデックスへ保存する。

## カテゴリ

- すべて
- スクショ
- 領収書候補
- 書類写真候補
- 名刺候補
- 看板候補
- ホワイトボード候補
- 工事写真候補
- 旅行写真候補
- 動画
- その他

## 推定方法

OCR前:

- 動画は `mediaType` で分類する
- スクリーンショットは `mediaSubtypes` で分類する
- 書類写真候補は画像の縦横比から低い信頼度で推定する
- 判定できないものはその他にする

OCR後:

- OCRテキスト内の語句からカテゴリを再推定する
- 領収書、名刺、ホワイトボード、工事、看板、旅行、書類の順に候補語句を見る
- 再推定後は `PhotoIndexService` 経由で `PhotoIndexStore` に保存する

## 保存項目

分類結果は `PhotoIndexRecord` に含める。

- `localIdentifier`
- `inferredCategory`
- `categoryConfidence`
- `categoryUpdatedAt`
- `lastSeenAt`

OCR済みの写真では、同じレコードに `ocrStatus` と `ocrText` も入り、OCRテキスト検索とカテゴリ再推定に使う。

## 信頼度

分類結果は `confidence` を持つ。

- 0.8以上: 高
- 0.5以上: 中
- 0.5未満: 低

詳細画面ではカテゴリ名と信頼度を表示する。

## UI

写真一覧と検索画面にカテゴリチップを表示する。選択したカテゴリで読み込み済み写真を絞り込む。

設定画面にはカテゴリ別件数、分類済み件数、検索インデックス件数を表示する。

詳細画面では、1枚の分類を再判定する操作と、未分類に戻す操作を用意する。設定画面では読み込み済み写真の分類を再構築できる。

分類のリセットは、しまい箱内の仮想フォルダ情報だけを対象にする。写真アプリ側の写真、アルバム、iCloud写真は変更しない。

OCR結果を削除した写真は、OCRテキストなしで軽量分類を再判定する。OCR由来の分類をもう一度使いたい場合は再OCRする。

## 安全方針

- 分類は仮想フォルダであり、写真アプリ側には反映しない
- 写真本体の移動、削除、リネーム、上書きはしない
- カテゴリは再推定できる内部メタデータとして扱う
- 検索インデックスには写真本体やサムネイル本体を保存しない
