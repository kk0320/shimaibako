# 自動分類と仮想フォルダ設計

## 目的

写真本体を動かさず、しまい箱内だけで写真を探しやすく分類する。分類結果は `localIdentifier` に紐づけて検索インデックスへ保存する。

## カテゴリ

- すべて
- 未分類
- スクショ
- 書類写真候補
- 領収書候補
- 名刺候補
- 看板候補
- ホワイトボード候補
- 工事写真候補
- 旅行写真候補
- 花・植物候補
- 桜・紅葉候補
- 建物・街並み候補
- 神社・寺・史跡候補
- 芸術品・展示候補
- 食べ物候補
- ペット・動物候補
- 人物写真候補
- 動画
- その他

花・植物、建物、芸術品、食べ物、動物、人物などはOCRだけでは判定しにくいため、低めの信頼度で「候補」として表示する。断定カテゴリとしては扱わない。

## スクショ特殊枠

スクショは通常写真とは別に、記録・メモ用途の仮想フォルダとして扱う。スクショを選択した時だけ、次の細分類チップを表示する。

- すべてのスクショ
- アイデア・メモ候補
- Web記事・調べ物候補
- 予約・チケット候補
- 地図・場所候補
- 買い物・領収候補
- アプリ設定・エラー候補
- チャット・SNS候補
- 仕事・資料候補
- その他スクショ

スクショ細分類も仮想フォルダであり、写真本体や写真アプリ側のアルバムは変更しない。チャット/SNSなどプライバシー性の高い内容も、OCRテキストを端末内でだけ参照する。

## 推定方法

OCR前:

- 動画は `mediaType` で分類する
- スクリーンショットは `mediaSubtypes` で分類する
- 書類写真候補は画像の縦横比から低い信頼度で推定する
- 判定できないものは未分類にする

OCR後:

- OCRテキスト内の語句からカテゴリを再推定する
- 領収書、名刺、ホワイトボード、工事、書類、看板、旅行、一般写真候補の順に候補語句を見る
- スクショは通常カテゴリとしてはスクショに固定し、OCRテキストでスクショ細分類を再推定する
- 再推定後は `PhotoIndexService` 経由で `PhotoIndexStore` に保存する

手動分類後:

- 詳細画面でユーザーがカテゴリやスクショ細分類を手動変更した写真では、手動分類を最優先する
- 手動分類は `PhotoIndexRecord` の `manualCategory` / `manualScreenshotSubcategory` に保存する
- 自動判定に戻した場合は手動分類を解除し、保存済みOCRテキストとメタデータから再判定する
- 分類傾向学習がオンの場合だけ、手動分類から軽量な学習例を作る
- 学習例は候補分類の補助であり、手動分類を上書きしない

スクショ細分類のキーワード例:

- アイデア・メモ候補: メモ、ToDo、やること、アイデア、note、idea、checklist
- Web記事・調べ物候補: http、https、検索、記事、ニュース、Safari、Wikipedia
- 予約・チケット候補: 予約、チケット、QR、入場、座席、ホテル、ticket、booking
- 地図・場所候補: 地図、経路、徒歩、駅、住所、map、route、station
- 買い物・領収候補: 注文、購入、合計、送料、領収、請求、Amazon、楽天、円
- アプリ設定・エラー候補: 設定、エラー、ログイン、パスワード、通知、権限
- チャット・SNS候補: LINE、メッセージ、DM、投稿、コメント、返信
- 仕事・資料候補: 会議、資料、請求、見積、契約、PDF、Excel、PowerPoint

## 保存項目

分類結果は `PhotoIndexRecord` に含める。

- `localIdentifier`
- `inferredCategory`
- `categoryConfidence`
- `categoryReason`
- `categoryUpdatedAt`
- `screenshotSubcategory`
- `screenshotSubcategoryConfidence`
- `screenshotSubcategoryReason`
- `screenshotSubcategoryUpdatedAt`
- `lastSeenAt`

OCR済みの写真では、同じレコードに `ocrStatus` と `ocrText` も入り、OCRテキスト検索とカテゴリ再推定に使う。

手動分類を持つ写真では、追加で次を保存する。

- `manualCategory`
- `manualScreenshotSubcategory`
- `manualCategoryUpdatedAt`

これらはしまい箱内の仮想フォルダ情報であり、写真本体や写真アプリ側のアルバムは変更しない。

## 信頼度

分類結果は `confidence` を持つ。

- 0.8以上: 高
- 0.5以上: 中
- 0.5未満: 低

詳細画面ではカテゴリ名、信頼度、分類理由、スクショ細分類、スクショ細分類理由を表示する。

## UI

写真一覧と検索画面にカテゴリチップを表示する。カテゴリチップは横スクロールで、すべて、未分類、スクショ、書類写真候補、領収書候補を前方に置く。選択したカテゴリで読み込み済み写真を絞り込む。

スクショを選択した時だけ、スクショ細分類チップを表示する。たとえば「スクショ > 地図・場所候補」「スクショ > 予約・チケット候補」のように絞り込める。

設定画面にはカテゴリ別件数、スクショ細分類件数、分類済み件数、検索インデックス件数を表示する。

詳細画面では、1枚の分類を再判定する操作と、未分類に戻す操作を用意する。設定画面では読み込み済み写真の分類を再構築できる。

分類のリセットは、しまい箱内の仮想フォルダ情報だけを対象にする。写真アプリ側の写真、アルバム、iCloud写真は変更しない。

OCR結果を削除した写真は、OCRテキストなしで軽量分類を再判定する。OCR由来の分類をもう一度使いたい場合は再OCRする。

詳細画面では分類を手動変更できる。手動変更した写真は「手動分類」と表示し、自動判定や学習由来候補より優先する。分類傾向学習がオンの場合、手動変更から端末内の軽量学習例を作る。

設定画面では分類傾向学習のオン/オフ、学習データ件数、学習データ削除を表示する。削除されるのは分類傾向データだけで、写真本体、OCR結果、手動分類は残る。

## 安全方針

- 分類は仮想フォルダであり、写真アプリ側には反映しない
- 写真本体の移動、削除、リネーム、上書きはしない
- カテゴリは再推定できる内部メタデータとして扱う
- カテゴリは断定ではなく候補として扱う
- 検索インデックスには写真本体やサムネイル本体を保存しない
- 手動分類がある写真では、手動分類を自動分類と学習由来候補より優先する
- 分類傾向学習はOCRキーワード、スクショ判定、メディア種別、縦横比の軽い加点だけに使う
- 画像特徴量比較や写真同士の類似度計算はこの段階では行わない
