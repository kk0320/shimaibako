# 実機確認手順

## 目的

しまい箱 iOS版をiPhone実機で起動し、精度向上モードと全体の削除/整理操作が安全に動くことを確認する。元写真・元動画、OCR結果、手動分類は意図しない削除・変更をしない。

## 事前確認

1. Macで作業ディレクトリを開く

```sh
cd ~/Desktop/all_dev/shimaibako
```

2. ビルドが通ることを確認する

```sh
xcodebuild build \
  -project ios/ShimaiBako/ShimaiBako.xcodeproj \
  -scheme ShimaiBako \
  -destination 'generic/platform=iOS Simulator'
```

3. 作業ツリーに意図しない `xcuserdata` や `UserInterfaceState` がないことを確認する

## テスト用写真・動画の準備

1. iPhoneの写真アプリに、確認用の写真を数枚用意する
2. 可能なら確認用の短い動画も1件用意する
3. 検証中は、写真アプリ側で元写真・元動画が残っているかを随時確認する
4. しまい箱の操作で写真アプリ側の写真、動画、アルバム、iCloud写真が削除、移動、変更されないことを確認する

## Xcodeで実機へ入れる

1. MacでXcodeを開く
2. `ios/ShimaiBako/ShimaiBako.xcodeproj` を開く
3. 実機iPhoneをUSBまたは同一ネットワークで接続する
4. Xcode上部の実行先から対象端末を選ぶ
5. 必要に応じてSigning Teamを確認する
6. Runする
7. iPhoneでしまい箱が起動することを確認する

## 精度向上モードの確認

1. アプリ下部タブから設定画面を開く
2. 「精度向上モード」が表示されることを確認する
3. 精度向上モードをONにする
4. 実行タイミングを「手動のみ」にする
5. アプリを終了して再起動し、ON/OFFと実行タイミングが残ることを確認する
6. 実行タイミングを「夜間に自動実行を試みる」に切り替え、保存されることを確認する
7. 説明文が「必ず指定時刻に実行」ではなく、iOSの判断で夜間に試みる内容になっていることを確認する

## 手動実行とキャンセル

1. 精度向上モードをONにする
2. 「今すぐ少しだけ実行」を押す
3. 対象件数が最大50件に収まることを確認する
4. 実行中に「キャンセル」を押す
5. 中断件数と中断理由が表示されることを確認する
6. 写真グリッド、検索、詳細画面が通常どおり開けることを確認する
7. OCR結果や手動分類が消えていないことを確認する
8. iPhoneの写真アプリを開き、元写真・元動画が削除、移動、変更されていないことを確認する

## 完了まで実行

1. 再度「今すぐ少しだけ実行」を押す
2. 完了まで待つ
3. 完了件数、失敗件数、中断件数、手動分類保護件数を確認する
4. 開始日時、終了日時、実行結果、実行モードが更新されることを確認する
5. アプリを終了して再起動する
6. 最終実行履歴と件数が残ることを確認する

## 手動分類優先の確認

1. 詳細画面で任意の写真を手動分類する
2. 設定画面に戻り、精度向上モードを実行する
3. 手動分類した写真の分類が自動再判定で上書きされないことを確認する
4. 必要に応じて手動分類保護件数が増えていることを確認する

## 精度向上データ削除の確認

1. 設定画面で「精度向上データを削除」を押す
2. 確認ダイアログで、削除対象が分類傾向学習データ、将来特徴量枠、処理履歴のみであることを読む
3. 削除を実行する
4. OCR結果が残っていることを詳細画面またはOCR検索で確認する
5. 手動分類が残っていることを詳細画面で確認する
6. iPhoneの写真アプリを開き、元写真・元動画、アルバム、iCloud写真が削除、移動、変更されていないことを確認する

## アプリ削除時の確認

1. 写真アプリにテスト用写真・動画が残っていることを確認する
2. しまい箱で読み込み、分類、OCR、検索を実行する
3. 必要なら手動分類を設定する
4. しまい箱アプリを削除する
5. 写真アプリを開く
6. テスト用の元写真・元動画が残っていることを確認する
7. しまい箱を再インストールする
8. しまい箱内のローカル分類、OCR結果、履歴は消えている可能性があることを確認する
9. 元写真・元動画は写真アプリ側に残っていることを再確認する

## 低電力モード確認

1. iPhoneの設定アプリで低電力モードをONにする
2. しまい箱を開く
3. 設定画面で低電力モードの注意が表示されることを確認する
4. 「今すぐ少しだけ実行」を押した場合、安全側で中断または実行抑制されることを確認する
5. 低電力モードをOFFに戻す

## 充電中/非充電時の確認

1. iPhoneを充電中にして設定画面を開く
2. バッテリー表示が充電中として読めることを確認する
3. iPhoneを充電から外す
4. バッテリー50%未満の場合、安全側で中断または実行抑制されることを確認する
5. バッテリー50%以上の場合、通常どおり手動実行できることを確認する

## iCloud取得モード確認

1. iCloud取得を「オフライン優先」にする
2. 説明文が端末内優先で読めることを確認する
3. iCloud取得を「iCloud取得を許可」にする
4. 通信量注意の説明が分かることを確認する
5. OCRや精度向上モードで元写真・元動画が外部OCRサービスへ送信されない方針が保たれていることを確認する

## 空き容量不足テスト

空き容量不足は無理に再現しない。端末の保存容量を意図的に圧迫すると、他のアプリや写真ライブラリに影響する可能性があるため、実機では表示文言とコード上の中断条件確認に留める。
