# 精度向上モード設計

## 目的

寝る前や充電中など端末状態が良い時に、OCR済み写真の再分類、未分類写真の軽量再判定、スクショ細分類、手動分類傾向の反映、検索インデックス整理を少しずつ進める。

3万枚規模でも端末が固まらないように、一度に処理する件数を絞る。写真本体は保存、送信、変更しない。

## 初期実装

初期実装は手動実行中心にする。設定画面に「今すぐ少しだけ実行」ボタンを置き、1回あたり最大50件だけ処理する。

処理内容:

- 未分類写真の軽量分類再判定
- OCR済み写真のカテゴリ再判定
- スクショ細分類の再判定
- 手動分類傾向の反映
- 検索インデックス整理

初期実装では新規OCRや画像特徴量生成は行わない。画像本体を読み込まず、保存済みメタデータとOCR結果、手動分類傾向だけで再判定する。

## 設定

設定画面で次を扱う。

- 精度向上モード: ON/OFF
- 実行タイミング
  - 手動のみ
  - 夜間に自動実行を試みる
- 推奨時間帯: 23:00〜6:00
- iCloud取得
  - オフライン優先
  - iCloud取得を許可

説明文:

「夜間や充電中に、分類の精度を少しずつ高めます。写真本体は外部送信せず、端末内で処理します。」

## 安全条件

実行前と処理中に端末状態を確認する。

- 充電中またはバッテリー50%以上
- 低電力モードOFF
- 発熱状態が通常またはやや高め
- 空き容量2GB以上推奨
- Wi-Fi推奨

中断条件:

- キャンセル操作
- 低電力モードON
- `thermalState` が `serious` または `critical`
- バッテリー50%未満かつ充電中ではない
- 空き容量1GB未満

中断時は完了済みの再判定だけ保存する。未処理分は次回以降に残す。中断そのものを写真単位の失敗として扱わない。

## 処理対象の選び方

読み込み済み写真から最大50件を選ぶ。

優先順:

1. 未分類写真
2. OCR済み写真
3. スクショ
4. その他の画像

手動分類がある写真では、手動分類を自動判定や分類傾向学習より優先する。

## 精度向上データ削除

設定画面から「精度向上データを削除」を実行できる。確認ダイアログを必ず表示する。

削除対象:

- 分類傾向学習データ
- 将来の画像特徴量データ
- 精度向上モードの処理履歴

削除しないもの:

- 写真本体
- OCR結果
- 手動分類
- 写真アプリ側の写真、アルバム、iCloud写真

## 画像特徴量の扱い

Vision Feature Printなど画像特徴量の本格処理は初期実装では行わない。

将来使う場合の方針:

- 花・植物、建物、芸術品、食べ物、動物などOCRで弱いカテゴリの補助に使う
- 未分類、手動分類済み、新規写真などに限定して試す
- 全3万枚を一気に処理しない
- 充電中、Wi-Fi、発熱なし、空き容量ありの場合だけ処理する
- 特徴量は端末内保存に限定する
- 写真本体は保存・送信しない
- 精度向上データ削除で消せるようにする
