# 手動分類傾向学習設計

## 目的

ユーザーが詳細画面で手動修正したカテゴリを、次回以降の候補分類の補助に使う。3万枚規模の写真ライブラリでも端末負荷を抑えるため、写真本体や画像特徴量は扱わず、OCRキーワードと軽いメタデータだけを保存する。

分類傾向学習は断定分類ではない。手動分類がある写真では、手動分類を自動分類や学習由来候補より常に優先する。

## 保存先

- `Application Support/ShimaiBako/manual_category_learning.json`
- Codable JSON
- 端末内のみ

検索インデックス本体の `photo_index.json` とは分けて保存する。将来SQLiteへ移行する場合は、別テーブルとして移せる構成にする。

## 保存する情報

`ManualCategoryLearningExample` に保存する。

- `sourceLocalIdentifier`
- `correctedCategory`
- `correctedScreenshotSubcategory`
- `normalizedKeywords`
- `isScreenshot`
- `mediaTypeRawValue`
- `aspectRatioBucket`
- `originalAutoCategory`
- `createdAt`
- `updatedAt`
- `useCount`

`normalizedKeywords` はOCRテキストから抽出した短いキーワードだけに制限する。1例あたり最大20個まで保存する。

## 保存しない情報

- 写真本体
- サムネイル本体
- 画像特徴量ベクトル
- 大量のOCR全文
- 全写真同士の類似度行列

分類傾向学習は、写真やサムネイルを読み込まずに使える軽量データだけを対象にする。

## 上限と整理方針

- 全体上限: 800件
- 1分類あたり上限: 80件
- 1例あたりキーワード上限: 20個
- 分類時の候補確認上限: 60件

上限を超えた場合は、`useCount` が少なく、`updatedAt` が古いものから整理する。同じ写真を何度も手動変更した場合は、重複作成せず同じ `sourceLocalIdentifier` の例を更新する。

## 学習例を作るタイミング

詳細画面で次の操作をした時だけ作る。

- 分類を手動変更
- スクショ細分類を手動変更

分類傾向学習がオフの場合、新しい学習例は作らない。既存の手動分類そのものは維持する。

自動判定に戻した場合は、その写真の手動分類を解除し、対応する学習例も削除する。

## 候補分類への使い方

分類時に全写真や画像本体は比較しない。次の軽い条件で候補を絞り、最大60件だけを見る。

- OCRキーワードが一致する
- スクショかどうかが一致する
- メディア種別が一致する
- 縦横比バケットが一致する

加点の考え方:

- OCRキーワード一致は強めに加点
- スクショ一致、メディア種別一致、縦横比一致は軽く加点
- 使われた回数は小さく加点

自動分類の信頼度が高い場合は、学習由来候補のスコアが十分高い時だけ反映する。学習由来で分類した場合も、詳細画面では「手動分類傾向を参考」と表示する。

## ON/OFF

設定画面で分類傾向学習をオン/オフできる。

オン:

- 手動分類から学習例を作る
- 既存学習例を候補分類の補助に使う

オフ:

- 新しい学習例は作らない
- 学習例を候補分類に使わない
- 既存の手動分類は維持する

## 学習データ削除

設定画面から学習データを削除できる。確認ダイアログを必ず表示する。

削除されるもの:

- `manual_category_learning.json` の分類傾向学習データ

削除されないもの:

- 写真本体
- サムネイル本体
- OCR結果
- 手動分類
- 検索インデックス

学習データを削除しても、手動分類済みの写真は手動分類のまま残る。

## 精度向上モードとの関係

精度向上モードでは、手動分類傾向を分類再判定の補助として使う。実行対象は1回最大50件に制限し、未分類写真、OCR済み写真、スクショなどを少しずつ再判定する。

精度向上モードが行うのは、保存済みメタデータ、OCR結果、手動分類傾向を使った軽量な再判定である。写真本体の読み込み、画像特徴量計算、全写真同士の比較は行わない。

設定画面の「精度向上データを削除」では、分類傾向学習データ、将来の画像特徴量データ、精度向上モードの処理履歴を削除する。削除しても写真本体、OCR結果、手動分類、写真アプリ側のデータは残る。

## 端末負荷への配慮

- 写真本体を読み込まない
- 画像特徴量を計算しない
- 画像同士の類似度比較をしない
- 全件再分類を自動実行しない
- 学習データ数と候補比較数に上限を設ける

3万枚規模では、まずメタデータ、OCRキーワード、手動分類履歴だけで軽く補助する。より重い画像特徴量ベースの分類は、実機でCPU、メモリ、バッテリー、発熱、保存容量を確認してから検討する。
