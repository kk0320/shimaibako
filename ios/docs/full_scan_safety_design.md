# 全件モード安全設計

## 目的

3万枚規模の写真ライブラリで、全件読み込みや大量OCRを不用意に開始しないようにする。

## 全件モードの確認フロー

読み込みモードで「大量」または「フル」を選んだ場合、すぐには適用せず安全確認画面を表示する。

表示する内容:

- バッテリー残量が50%以上、または充電中に使用してください
- 大量の写真を処理すると端末が発熱する場合があります
- iCloud写真を取得する場合は通信量が増える場合があります
- 写真本体は変更しませんが、大規模処理前は念のためバックアップを推奨します
- 処理中はメモリを使用するため、他のアプリの動作が重くなる場合があります
- 保存容量が不足している場合、処理を中断します
- 最初は軽量/標準モードで試すことを推奨します

ボタン:

- 軽量モードに戻る
- 理解して全件モードを使う

端末温度が高い場合、または保存容量が1GB未満の場合は続行を無効にする。

## 端末状態チェック

`DeviceSafetyService` で次を確認する。

- バッテリー残量
- 充電状態
- 低電力モード
- 発熱状態
- 保存容量

目安:

- バッテリー50%未満かつ充電中でない場合は警告
- 低電力モード中は大量OCRを推奨しない
- `thermalState` が `serious` または `critical` の場合は大量処理を開始しない
- 空き容量1GB未満では開始しない
- 空き容量2GB以上を推奨する

## まとめてOCR安全ゲート

まとめてOCR前に確認画面を表示する。

- OCRは端末内で実行される
- 写真は外部送信しない
- 処理中は発熱やバッテリー消費が増える場合がある
- iCloud上の写真は取得に時間がかかる場合がある
- 途中でキャンセルしても、完了済みのOCR結果は保存される

対象は最大20件に制限する。候補が100件以上ある場合でも、一度に全件OCRは行わない。

## 自動中断

まとめてOCR中に次を検知した場合は中断する。

- キャンセル操作
- アプリがバックグラウンドへ移行
- `thermalState` が `serious` または `critical`
- 空き容量1GB未満

中断時:

- 完了済みのOCR結果は保存する
- 未開始の写真は未処理のまま残す
- 中断理由を画面に表示する
- キャンセルや中断そのものを写真単位の失敗として保存しない

## 精度向上モード安全ゲート

精度向上モードの手動実行でも、大量処理と同じ考え方で安全条件を確認する。初期実装では1回最大50件に制限し、全ライブラリを自動で処理しない。

開始前と処理中に確認する内容:

- バッテリー50%以上、または充電中
- 低電力モードOFF
- 発熱状態が `normal` または `fair`
- 空き容量1GB以上、2GB以上を推奨
- iCloud取得を許可している場合は通信量に注意

中断条件:

- キャンセル操作
- `thermalState` が `serious` または `critical`
- 低電力モード
- バッテリー50%未満かつ充電中ではない
- 空き容量1GB未満

中断時は、完了済みのインデックス更新だけを残し、未処理分は次回以降に回す。写真本体は保存、送信、変更しない。

「夜間に自動実行を試みる」は将来のバックグラウンド処理設計であり、iOSの判断に依存する。指定時刻に必ず実行されるものではない。

## 運用方針

3万枚規模では、初回から全件OCRを実行しない。

推奨順:

1. 軽量または標準モードで写真アクセスと表示を確認する
2. メタデータと軽量分類で仮想フォルダを確認する
3. スクショ、書類候補、表示中などに絞ってOCRする
4. 必要に応じて読み込みモードを多め、大量へ広げる
