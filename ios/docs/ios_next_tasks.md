# iOS次タスク

## 優先度高

1. 実機画面で写真権限フローを確認する
2. 限定アクセスで選択写真だけが表示されることを確認する
3. 3万枚規模の実機ライブラリで軽量/標準/多め/大量モードの表示負荷を確認する
4. 大量/フルモードの安全確認画面を実機で確認する
5. 実機でOCR結果の保存と再起動後の復元を確認する
6. 実機でまとめてOCR中のキャンセル、連続タップ、バックグラウンド移行時の挙動を確認する
7. iCloud取得を許可した場合とオフライン優先の場合のOCR失敗表示を確認する

## 優先度中

1. UIテストターゲットを追加する
2. 写真グリッドの空状態と読み込み状態を追加検証する
3. スクリーンショット判定の精度を上げる
4. 動画サムネイル表示と種別バッジを追加確認する
5. SettingsViewにバージョン情報を追加する
6. OCR結果キャッシュ削除UIを設計する
7. カテゴリ推定の辞書を実機写真に合わせて調整する
8. 写真詳細画面のメタデータ表示を増やす

## 優先度低

1. 写真枚数が多い場合のページングまたは段階読み込みを追加する
2. `PhotoIndexStore` と `OCRResultStore` をSQLiteまたはSwiftDataへ移行する
3. 追加ファイル種別への拡張方針を決める
4. OCR認識言語と精度をユーザーが変更できる形にする
5. OCR処理履歴を見られるようにする

## 注意点

- 写真は外部送信しない
- 写真の削除、移動、リネーム、上書き機能は作らない
- 写真アプリ側にアルバムやフォルダを作らない
- 3万枚規模では初回から全件OCRを行わない
- まずメタデータと軽量分類を使い、OCRは表示中、スクショ、書類候補などに絞って段階実行する
- 外部APIモードは次段階以降に回す
- App Store提出、証明書、課金関連はこの段階では扱わない
- テスト用の起動引数はDebugビルドのみで使う

## 実機確認メモ

実機へのビルド、インストール、起動は確認済み。以下は端末画面での手動確認として残る。

1. 写真アクセスを未確認から許可へ進める
2. 限定アクセスで数枚だけ選び、グリッドが選択範囲だけになることを確認する
3. 設定画面の「写真の選択を変更」から選択範囲を変更する
4. 1枚OCRして結果が詳細画面に表示されることを確認する
5. アプリを終了して再起動し、OCR結果が残ることを確認する
6. OCR結果に含まれる語句で検索し、対象写真が表示されることを確認する
7. まとめてOCRを開始し、キャンセル後に完了済みだけ保存されることを確認する
8. 読み込みモードを標準、多め、大量へ切り替え、設定が保存されることを確認する
9. 大量/フルモード選択時に安全確認画面が表示されることを確認する
10. iCloud取得モードを切り替え、端末内未取得写真で失敗表示またはOCR実行を確認する
