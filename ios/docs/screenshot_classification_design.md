# スクショ細分類設計

## 目的

スクリーンショットを通常写真とは別の「記録・メモ」用途として扱い、しまい箱内の仮想フォルダで探しやすくする。写真本体は移動、削除、変更しない。写真アプリ側にアルバムやフォルダも作らない。

## サブカテゴリ

- すべてのスクショ
- アイデア・メモ候補
- Web記事・調べ物候補
- 予約・チケット候補
- 地図・場所候補
- 買い物・領収候補
- アプリ設定・エラー候補
- チャット・SNS候補
- 仕事・資料候補
- その他スクショ

すべて候補分類であり、断定として扱わない。

## 判定方法

スクショ判定はPhotoKitの `mediaSubtypes` に含まれるスクリーンショット情報を優先する。分類対象がスクショの場合だけ、OCRテキストからサブカテゴリを推定する。

主なキーワード:

- アイデア・メモ候補: メモ、TODO、ToDo、やること、アイデア、note、idea、checklist
- Web記事・調べ物候補: http、https、www、検索、記事、ニュース、Safari、Wikipedia、browser
- 予約・チケット候補: 予約、チケット、QR、入場、座席、便、ホテル、confirmation、ticket、booking
- 地図・場所候補: 地図、経路、徒歩、駅、住所、map、route、station
- 買い物・領収候補: 注文、購入、合計、送料、領収、請求、Amazon、楽天、円、order、receipt
- アプリ設定・エラー候補: 設定、エラー、ログイン、パスワード、通知、権限、error、settings、login
- チャット・SNS候補: LINE、メッセージ、DM、投稿、コメント、返信、chat、message、post
- 仕事・資料候補: 会議、資料、請求、見積、契約、PDF、Excel、PowerPoint、meeting、document

該当しない場合は「その他スクショ」とする。

## 保存項目

`PhotoIndexRecord` に次の項目を保存する。

- `screenshotSubcategory`
- `screenshotSubcategoryConfidence`
- `screenshotSubcategoryReason`
- `screenshotSubcategoryUpdatedAt`

既存の `photo_index.json` にこれらの項目がなくても読み込める。保存先は端末内の検索インデックスだけで、写真本体やサムネイル本体は保存しない。

## UI

写真一覧と検索画面では、カテゴリで「スクショ」を選んだ時だけサブカテゴリチップを表示する。

例:

- スクショ > 地図・場所候補
- スクショ > 予約・チケット候補
- スクショ > アイデア・メモ候補

詳細画面では、推定カテゴリ、信頼度、分類理由、スクショ細分類、スクショ細分類理由を表示する。設定画面ではスクショ細分類ごとの件数を表示する。

詳細画面ではスクショ細分類を手動変更できる。手動変更したスクショでは、手動細分類をOCRキーワードによる自動細分類より優先する。自動判定に戻すと、保存済みOCRテキストとメタデータから再判定する。

分類傾向学習がオンの場合、手動で直したスクショ細分類から軽量な学習例を作る。学習例はOCRキーワード、スクショ判定、メディア種別、縦横比だけを使い、写真本体やサムネイル本体は保存しない。

## プライバシー

チャット/SNS、予約、仕事資料などプライバシー性の高い内容も端末内で処理する。外部OCRサービス、外部API、外部クラウドには送信しない。

分類傾向学習でも、保存するOCR由来情報は正規化した短いキーワードだけに制限する。大量のOCR全文、画像特徴量、写真同士の類似度行列は保存しない。
