# フル読み込み進捗設計

## 目的

3万枚規模の写真ライブラリでも、フル/全件読み込みで白画面や永久読み込みに見えないようにする。元写真・元動画は削除・変更しない。

## 読み込み方式

- フル/全件でも一括で最後まで待たず、100件単位で画面へ反映する
- 読み込み開始時に進捗を保存する
- 長時間読み込みジョブは `PhotoLibraryService` が保持し、SwiftUI View の `.task` や `onAppear` に直接依存させない
- 写真タブから検索タブ、設定タブへ移動しても読み込みは継続する
- まず直近100件を表示し、その後もバッチごとに追加する
- 各バッチ後にUIへ件数と進捗率を表示する
- キャンセルしても、すでに読み込んだ写真参照と既存インデックスは壊さない
- 起動直後に重い処理を同期実行しない
- 写真グリッドは初期200件を表示し、「さらに表示」で200件ずつ増やす
- View再表示時の検索インデックス再構築は、最新バッチまたは表示中ページ相当に制限する

Viewは開始、キャンセル、再開の指示と進捗表示だけを担当する。タブ移動やView再生成で読み込みTaskが破棄されても、それをユーザー操作のキャンセルとして扱わない。

## 進捗状態

進捗はUserDefaultsに軽量保存する。

- 待機中
- 写真一覧を取得中
- インデックス作成中
- サムネイル準備中
- 完了
- キャンセル
- 失敗
- 前回の読み込みが途中で停止
- 一時停止

表示する情報は、状態、件数、進捗率、最終更新時刻、経過時間、キャンセル/復旧ボタンである。実機調査用に、個人情報を含まない範囲で最終成功件数、直近フェーズ、メモリ警告回数も表示する。

保存する進捗情報は、読み込みモード、読み込み済み件数、総件数、開始/更新/終了時刻、中断理由、直近バッチの `localIdentifier`、直近バッチ範囲、最終成功件数、フェーズ名に限定する。写真本体、サムネイル本体、OCR全文は進捗情報に保存しない。

中断理由は区別して保存する。

- ユーザー操作で中止しました
- 画面切替では読み込みを継続しています
- アプリの状態変化により一時停止しました
- 読み込みタスクが中断されました。続きから再開できます
- 端末のメモリ不足により一時停止しました

ユーザーがキャンセルボタンを押した場合だけ、ユーザーキャンセルとして表示する。

## stale job 判定

前回の進行中状態がアプリ起動時に残っている場合、その読み込みジョブは実際には存在しないため、復旧可能な停止状態として扱う。

実行中でも最終進捗更新から3分以上経過した場合は、進んでいない可能性をUIへ表示する。

## 復旧操作

復旧操作は読み込みジョブ状態だけを対象にする。

- 読み込みを中止
- 続きから再開
- 読み込み状態をリセット
- 軽量モードで再読み込み

これらの操作では、元写真・元動画、OCR結果、手動分類、不要候補/非表示状態、メモ、タグ、学習データ、精度向上履歴は削除しない。

## タブ移動とバックグラウンド

タブ移動はアプリ内の画面切替であり、読み込み停止条件にしない。進捗カードは写真画面と設定画面の両方で確認できる。

アプリがホーム画面やロック画面へ移った場合は、`UIApplication.beginBackgroundTask` で短時間だけバッチ完了や進捗保存を続ける。iOSから終了通知を受けた場合は進捗を保存し、`アプリの状態変化により一時停止しました` として続きから再開できる状態にする。BGTaskScheduler/BGProcessingTask は将来検討とし、初期実装では必須にしない。

メモリ警告を受けた場合は、サムネイルキャッシュとPhotoKitキャッシュを解放し、読み込みを一時停止する。完了済みバッチとSQLiteインデックスは残し、続きから再開できる状態にする。

## インデックス保護

同じ `localIdentifier` の既存データは保持する。

- displayState
- 手動分類
- OCR結果
- メモ
- タグ
- スクショ細分類

読み込みで追加されるのは、PhotoKitから読んだメタデータとPHAsset参照である。写真本体はアプリDBに保存しない。

大規模モードでは、SwiftUIへ全件分の `PhotoAsset` を渡さない。表示用に保持する `PhotoAsset` は最大500件程度に制限し、グリッド表示ではSQLiteのIDページから必要な分だけPhotoKitで復元する。
