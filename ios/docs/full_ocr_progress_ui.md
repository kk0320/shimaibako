# OCR進捗UI

## 方針

全数OCRの進捗カードは通常UIから外す。リリース前の通常機能では、写真タブに `まとめてOCR（最大2,000件）` のコンパクトな進捗だけを表示する。

過去の検証で残った全数OCRジョブ状態は、元写真・元動画、OCR結果、検索インデックスを消さずに非表示または無効扱いにする。

## まとめてOCR進捗

写真タブでは次を表示する。

- 対象件数
- 完了件数
- 処理中件数
- 失敗件数
- キャンセル済みまたは中断理由

通常時はコンパクト表示にし、詳細は展開時だけ表示する。写真一覧、検索欄、カテゴリチップの操作領域を優先する。

## 非表示にするもの

通常UIでは次を表示しない。

- 全数OCR管理
- 全数OCR確認
- 全数OCR実行中カード
- 全数OCR完了カード
- `0 / 0件` の壊れた完了カード
- ライブラリ全体を対象にしたOCR開始導線

## 旧ジョブ状態

`ocr_jobs.sqlite` に古い全数OCRジョブが残っていても、通常UIには流さない。

- `totalCount == 0` の完了ジョブは無効扱い
- staleな全数OCRジョブは停止済み扱い
- `OCRProgressStore.activeSnapshot` へ出さない
- 既存OCR結果は保持する
- 既存SearchDocumentは保持する

## 安全表示

まとめてOCRの画面には、次を明記する。

- OCRは端末内で実行される
- 写真は外部送信しない
- 元写真・元動画は削除・変更しない
- iCloud上の写真は取得に時間がかかる場合がある
- キャンセルしても完了済みOCR結果は保存される

## Debug

Debug用のOCR診断表示は、設定画面で明示的にONにした場合だけ表示する。通常のSimulator確認や実機確認では非表示にする。

Debug用テストデータやダミーOCRは、PhotoKitへ書き込まず、元写真・元動画には触れない。
