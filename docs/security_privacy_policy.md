# セキュリティ・プライバシー方針

## ローカルファースト

しまい箱 はローカルMVPとして動作します。写真フォルダをPC上で読み取り、検索インデックスとサムネイルをローカルの `data/` 配下に保存します。

## 読み取り専用

スキャン処理はファイルの読み取りだけを行います。元写真、元動画、元ファイルに対して、削除、移動、リネーム、上書きは行いません。

## 外部送信なし

写真本体、動画本体、ファイルパス、EXIFなどのメタデータを外部送信しません。検索、サムネイル生成、重複候補検出はPC内で処理します。

## 外部検証画像

`data/external_test_assets` は、自分のスマホ写真を使わずに実写真風の検証を行うための追加データセットです。Openverseでライセンス情報が明確な候補から取得し、`manifest.json` と `manifest.csv` に出典URL、出典サイト、ライセンス名、作者/クレジット、同梱可否を保存します。

外部検証画像の取得時も、ユーザーの写真本体や生成済みサムネイル、OCR結果、ローカルDBを外部送信しません。Google画像検索などから無作為に拾った画像は使いません。

NC/ND付きライセンスなど利用条件が複雑になりやすい画像は `allowed_for_demo_bundle=false` として扱います。デモ配布へ同梱する場合は、必ずmanifestの出典URLとライセンス条件を確認してください。

## OCR

OCRはユーザーが明示的に開始した場合だけ実行します。全写真を自動でOCRすることはありません。OCRはローカルの `tesseract.exe` が検出できる場合にPC内で実行します。外部OCR API、有料API、クラウドOCRには接続しません。

OCRテキストは `data/app.db` に保存されます。DBバックアップにもOCRテキストが含まれるため、バックアップファイルも写真メタデータと同じ扱いで管理してください。

`data/test_assets` では、manifestの正解テキストを使うテスト用フォールバックを利用できます。これはテスト画像専用であり、実写真OCRの評価結果ではありません。

実写真でOCRを試す場合は、元写真フォルダではなく、`data/real_test_photos` などの検証用フォルダへコピーした写真だけを対象にしてください。最初は100〜300件程度に制限して、スキャンとOCRの動作を確認してください。

Tesseractの言語データは `data/tessdata` に保存できます。このフォルダはOCR用のローカルデータ置き場であり、写真本体ではありません。OCR実行時は `--tessdata-dir data/tessdata` 相当の指定でローカル処理します。

## 外部クラウドAPI接続なし

現在のMVPでは Microsoft Graph、Google API、Apple API、画像認識クラウドAPIへ接続しません。iCloud写真やOneDrive写真は、Windows上に同期済みのローカルフォルダとして扱います。

## 有料APIなし

有料API、有料クラウド、有料リソースは使用しません。

## 将来的なAPI接続

将来クラウド連携を追加する場合も、初期状態はOFFにします。ユーザーが明示的にONにした場合だけ接続し、接続先、保存される情報、取り消し方法を画面とドキュメントで明示します。

## APIキーやトークンの保存方針

将来APIキーやログイントークンを扱う場合は、平文保存しません。Windows Credential Manager、iOS Keychain など、OSの安全な保管機能を優先します。

## 家庭内LAN利用時の注意

デフォルト起動は `LocalOnly` モードです。backend / frontend は `127.0.0.1` にだけバインドし、PC内からのみ利用します。スマホ用URLは表示しません。

スマホからアクセスする場合だけ、`Start-ShimaiBako-LAN.cmd` で起動します。この場合、PC上の開発サーバーを家庭内LANに公開します。同じWi-Fi上の端末からアクセスできる可能性があるため、信頼できる家庭内ネットワークでのみ利用してください。

`LanAccess` モードでは、起動時にランダムPINを生成してPowerShell画面に表示します。PIN入力前は、検索、詳細、サムネイル、データソース、統計、OCR、ログ、設定などの主要APIにアクセスできません。`/api/health` は公開されますが、ファイルパスやスキャン詳細は返しません。

Windows Firewallで Python や Node.js の通信許可が必要になる場合があります。許可する場合も、家庭内LANなど信頼できるネットワークだけで利用してください。

## 公共Wi-Fiで使わない

公共Wi-Fi、ホテル、カフェ、共有オフィスなどのネットワークでは使わないでください。必要な場合は、PC側でローカルホストのみの起動に戻してください。

## 外部公開しない

ルーターのポート開放、外部公開トンネル、共有URL化は推奨しません。このMVPは家庭内LANまたはPC内利用を前提にしています。実写真を扱う前に、`LocalOnly` で使うか、`LanAccess` のPIN認証画面が表示されることを確認してください。
